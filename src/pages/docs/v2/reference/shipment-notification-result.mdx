# ShipmentNotificationResult Object

ShipmentNotificationResult objects are used throughout the Order Source specifications to describe the result of an individual shipment notification request.

<Reference>

{' '}

<Field name="notification_id" type="string" required={true}>
  <Description>The notification id from the request.</Description>
</Field>

{' '}

<Field name="succeeded" type="boolean" required={true}>
  <Description>This property has been deprecated, please use `status`.</Description>
</Field>

{' '}

<Field name="status" type="string" required={false}>
  <Description>Indicates the status for the third party receiving this notification.</Description>
</Field>

{' '}

<Field name="confirmation_code" type="string" required={false}>
  <Description>An optional confirmation code associated with this notification.</Description>
</Field>

{' '}

<Field name="failure_reason" type="string" required={false}>
  <Description>If `succeeded` is `false` use this field to indicate why.</Description>
</Field>

{' '}

<Field name="submission_id" type="string" required={false}>
  <Description>
    A unique identifier that is used to query the third party about the status of this notification in the
    notification_status endpoint.
  </Description>
</Field>

</Reference>
