`VerifyDeliveryOptions()`
===========================
This method is called when a user is initiating the registration process to add delivery options.

-----------------------------------------------

```javascript
  // Your code here
```

```typescript
  // Your code here
}
```
</CodeWrapper>



Parameters
-----------------------------------------------

### `VerifyDeliveryOptionsRequest`
An object containing parameters to use when registering delivery options with an order source. Currently there are no VerifyDeliveryOptions parameters.
Implemenatations for this method should just call to the order source to verify delivery options if possible.

Return Value
-----------------------------------------------

### `OrderAcknowledgementResponse`

<Reference>

  <Field name="auth" nullable={false}>
    <Type>
      [auth object](#auth)
    </Type>
    <Description>
      Represents the auth information sent with every request
    </Description>
  </Field>

  <Field name="is_supported" type="string" required={true}>
    <Description>
      Indicates whether or not delivery options is supported in the order source. 
      Cases where this might not always be true for every merchant happen when the order source has placed limitations on the order source account to enable delivery options.
      Examples would be the delivery options feature is locked behind a specific plan level or additional setup must be done by the merchant to enable it in the order source.
    </Description>
  </Field>

</Reference>
