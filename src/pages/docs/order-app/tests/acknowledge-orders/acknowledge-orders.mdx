Order App Tests
==================================

### `acknowledgeOrders`

This test exercises your app's ability to successfully acknowledge orders.


Test Parameters
-------------------------
This test has many parameters [that can be customized](#customizing-parameters) in your [`connect.config.js` file](../../../testing/config.mdx). Any parameters you don't override will use their default values.

<Reference>

  <Field name="notifications">
    <Type>
      object[]
    </Type>
    <Description>
        An array of objects representing order notifications.
    </Description>
    <DefaultValue>
        An array populated with the values below
    </DefaultValue>
  </Field>

  <Field name="notifications[].id" type="string">
    <Description>
        The marketplace's unique ID for the sales order.
    </Description>
    <DefaultValue>
        `989798798`
    </DefaultValue>
  </Field>

  <Field name="notifications[].identifiers" type="object">
    <Description>
        Your own identifiers for this sales order.
    </Description>
    <DefaultValue>
        `lksldm`
    </DefaultValue>
  </Field>

  <Field name="notifications[].orderNumber" type="string">
    <Description>
        The customer facing identifier of the sales order.
    </Description>
    <DefaultValue>
        `987987987`
    </DefaultValue>
  </Field>

  <Field name="notifications[].importedDate">
    <Type>
      [date/time object](./../../../reference/date-time.mdx)
    </Type>
    <Description>
        The date that the order was imported.
    </Description>
    <DefaultValue>
        Today at 6:00pm
    </DefaultValue>
  </Field>

</Reference>
