Order App Tests
==================================

### `acknowledgeOrders`

This test exercises your app's ability to successfully acknowledge orders.


Test Parameters
-------------------------
This test has many parameters [that can be customized](#customizing-parameters) in your [`connect.config.js` file](../../../testing/config.mdx). Any parameters you don't override will use their default values.

<Reference>

  <Field name="notifications" type="array">
    <Description>
        An array of `notification` objects which contain an `id` string, an `identifiers` object, an `orderNumber` and `importedDate`.
    </Description>
    <DefaultValue>
        An array populated with the values below
    </DefaultValue>
  </Field>

  <Field name="notifications[].id" type="string">
    <Description>
        A string `id`.
    </Description>
    <DefaultValue>
        `989798798`
    </DefaultValue>
  </Field>

  <Field name="notifications[].identifiers" type="object">
    <Description>
        An object filled with `id: "lksldm"`  or some other string key-value pair.
    </Description>
    <DefaultValue>
        `lksldm`
    </DefaultValue>
  </Field>

  <Field name="notifications[].orderNumber" type="string">
    <Description>
        A string order number.
    </Description>
    <DefaultValue>
        `987987987`
    </DefaultValue>
  </Field>

  <Field name="notifications[].importedDate" type="string">
    <Description>
        A complete ISO 8601 date/time with a time zone, or `Datetime` object.
    </Description>
    <DefaultValue>
        Today at 6:00pm
    </DefaultValue>
  </Field>

</Reference>
