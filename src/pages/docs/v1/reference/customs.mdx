# Customs Object

A customs object is used to represent customs information for international shipments.

## Properties

This table lists the properties of a customs object and identifies those properties that are required. The `nullable`
column indicates which properties may be null when the object is provided as an argument to one of your methods, and the `required`
column indicates which properties are required when the object is returned from one of your methods.

<Reference>

{' '}

<Field name="customs" type="object" nullable={false}>
  <Description>Customs declarations for this package.</Description>
</Field>

{' '}

<Field name="customs.nonDeliveryOption" type="string" nullable={true}>
  <Description>
    Indicates what should be done if the package cannot be delivered. If `undefined`, the default behavior of the
    receiving country's customs department applies, which may incur charges. Valid values include the following: *
    `return` - Return to sender. * `abandon` - Treat as abandoned.
  </Description>
</Field>

  <Field name="customs.contents" type="object[]" nullable={false}>
    <Description>
      Describes the contents of the package for customs purposes.

      Note that Customs contents may not correspond one-to-one with the package contents.
      Package contents usually include one item per unique merchandise SKU
      e.g. one red t-shirt and one blue t-shirt), whereas customs contents are often grouped by
      product type (e.g. two t-shirts). In addition, some package contents don't need to be declared
      for customs purposes.
    </Description>

  </Field>

{' '}

<Field name="customs.contents[].type" type="string" nullable={false}>
  <Description>
    The customs type category. Valid values include the following: * `gift` - The package contains a gift * `documents`
    - The package contains documents. * `merchandise` - The package contains merchandise. * `returned_goods` - The
    package is a returned shipment. * `commercial_sample` - The package contains a commercial sample, such as a flooring
    sample. * `humanitarian_donation` - The package contains a donation. * `other` - The package contains items that
    fall outside the other categories.
  </Description>
</Field>

{' '}

<Field name="customs.contents[].sku" type="string" nullable={false}>
  <Description>
    The [Stock Keeping Unit](https://en.wikipedia.org/wiki/Stock_keeping_unit). This string will not contain newline
    characters.
  </Description>
</Field>

{' '}

<Field name="customs.contents[].description" type="string" nullable={false}>
  <Description>
    A description of the item. Usually required if `type` is `other`. This string will not contain newline characters.
  </Description>
</Field>

{' '}

<Field name="customs.contents[].quantity" type="object" nullable={false}>
  <Description>The quantity of items in the package.</Description>
</Field>

{' '}

<Field name="customs.contents[].quantity.value" type="number" nullable={false}>
  <Description>The value of the quantity. The minimum value is `1`.</Description>
</Field>

{' '}

<Field name="customs.contents[].unitValue" type="object" nullable={false}>
  <Description>The monetary value of each unit in the package.</Description>
</Field>

{' '}

<Field name="customs.contents[].unitValue.value" type="number" nullable={false}>
  <Description>The amount of this value.</Description>
</Field>

{' '}

<Field name="customs.contents[].unitValue.currency" type="string" nullable={false}>
  <Description>The currency that the value represents.</Description>
</Field>

{' '}

<Field name="customs.contents[].countryOfOrigin" nullable={true}>
  <Type>[country code string](./country-codes.mdx)</Type>
  <Description>
    The [ISO 3166 country code](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_codes) for the
    [country](./country-codes.mdx) of origin. This is usually the same as `countryOfManufacture`, but some countries
    distinguish between the two for certain products.
  </Description>
</Field>

{' '}

<Field name="customs.contents[].countryOfManufacture" nullable={true}>
  <Type>[country code string](./country-codes.mdx)</Type>
  <Description>
    The [ISO 3166 country code](https://en.wikipedia.org/wiki/List_of_ISO_3166_country_code) for the
    [country](./country-codes.mdx) where the item was manufactured.
  </Description>
</Field>

{' '}

<Field name="customs.contents[].harmonizedTariffCode" type="string" nullable={false}>
  <Description>
    The [Harmonized Tariff Code](https://hts.usitc.gov/) for the item. This string must not contain newline characters.
  </Description>
</Field>

{' '}

<Field name="customs.buyerShippingAmountPaid" type="object" nullable={true}>
  <Description>
    Indicates how much the buyer paid for shipping, if any. This amount may be different than the shipping cost billed
    to the shipper.
  </Description>
</Field>

{' '}

<Field name="customs.buyerShippingAmountPaid.value" type="number" nullable={false}>
  <Description>The amount of this value.</Description>
</Field>

{' '}

<Field name="customs.buyerShippingAmountPaid.currency" type="string" nullable={false}>
  <Description>The currency that the value represents.</Description>
</Field>

{' '}

<Field name="customs.dutiesPaid" type="object" nullable={true}>
  <Description>
    The amount of duties paid by the shipper, if any. This is generally only necessary for DDP (bill duties to sender)
    shipments.
  </Description>
</Field>

{' '}

<Field name="customs.dutiesPaid.value" type="number" nullable={false}>
  <Description>The amount of this value.</Description>
</Field>

{' '}

<Field name="customs.dutiesPaid.currency" type="string" nullable={false}>
  <Description>The currency that the value represents.</Description>
</Field>

</Reference>
