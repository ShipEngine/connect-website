Parameters
-----------------------------------------------

### `SalesOrdersExportRequest`
An object containing parameters to use when querying the order source for sales orders.

<Reference>

  <Field name="criteria" type="object" nullable={true}>
    <Description>
      This object will contain properies that help create the search query to poll the order source for sales orders.
    </Description>
  </Field>

  <Field name="criteria.from_date_time" type="string" nullable={true}>
    <Description>
      The datetime associated with the beginning of the range that the request should pull orders from.
      Formatted per the https://tools.ietf.org/html/rfc3339 spec. Will always be in UTC. 
    </Description>
  </Field>

  <Field name="criteria.to_date_time" type="string" nullable={true}>
    <Description>
      The datetime associated with the end of the range that the request should pull orders from.
      Formatted per the https://tools.ietf.org/html/rfc3339 spec. Will always be in UTC. 
    </Description>
  </Field>

  <Field name="cursor" type="string" nullable={true}>
    <Description>
      Information needed to get a current batch of sales orders if necessary. This should be set by SalesOrdersExportResponse
      if there are additional orders to be retrieved.
    </Description>
  </Field>

  <Field name="sales_order_field_mappings" type="object" nullable={true}>
    <Description>
      Allows for mapping of user defined fields to properies on the SalesOrderExportResponse.
    </Description>
  </Field>

  <Field name="sales_order_field_mappings.custom_field_1" type="string" nullable={true}>
    <Description>
      Custom field 1
    </Description>
  </Field>

    <Field name="sales_order_field_mappings.custom_field_2" type="string" nullable={true}>
    <Description>
      Custom field 2
    </Description>
  </Field>

    <Field name="sales_order_field_mappings.custom_field_3" type="string" nullable={true}>
    <Description>
      Custom field 3
    </Description>
  </Field>

  <Field name="sales_order_status_mappings" type="object" nullable={true}>
    <Description>
      Merchant specified mappings for custom statuses coming from the 3rd parties api and the SalesOrderStatus they should map to. The keys of this object will be the custom status strings provided by a user, and the values will be one of the following valid status codes: 
        * `AwaitingPayment` - The sales order is awaiting payment.
        * `AwaitingShipment` - This sales order is awaiting shipment.
        * `OnHold` - This sales order is on hold.
        * `PendingFulfillment` - This sales order is pending fulfillment.
        * `Completed` - This sales order is completed.
        * `Cancelled` - This sales order has been cancelled.
    </Description>
  </Field>

</Reference>
