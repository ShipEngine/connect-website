# Billing Line Item Object

A billing line item object is used to represent billing line items on shipments.

## Properties

This table lists the properties of a billing line item object and identifies those properties that are required.

<Reference>

{' '}

<Field name="billing_category" required={true}>
  <Type>[billing category](#billing-categories)</Type>
  <Description>
    The [billing category](#billing-categories) for this line item.
  </Description>
</Field>

{' '}

<Field name="carrier_description" type="string" required={false}>
  <Description>
    If the carrier provides a description from their API about the billing
    charge (not specific to the user). Maximum length of 100 characters.
  </Description>
</Field>

{' '}

<Field name="carrier_billing_code" type="string" required={false}>
  <Description>
    If the carrier provides a billing code for the billing charge.
  </Description>
</Field>

{' '}

<Field name="memo" type="string" required={false}>
  <Description>
    This will often be empty; however, if there was additionally specific
    information about the charge it should go here, could also be a reference
    number. Maximum length of 250 characters.
  </Description>
</Field>

{' '}

<Field name="amount" type="object" required={true}>
  <Description>Total amount and currency of the line item.</Description>
</Field>

{' '}

<Field name="amount.amount" type="string" required={true}>
  <Description>The amount of the line item.</Description>
</Field>

{' '}

<Field name="amount.currency" type="string" required={true}>
  <Description>The currency of the line item.</Description>
</Field>

</Reference>

## Billing Categories

This table lists the valid values for billing categories.

<Reference>

{' '}

<Field name="uncategorized">
  <Description>
    Use this for any fees that don't align with the specific categories below.
  </Description>
</Field>

{' '}

<Field name="shipping">
  <Description>The base charge for the shipping amount.</Description>
</Field>

{' '}

<Field name="insurance">
  <Description>Fee for carrier insurance added to the shipment.</Description>
</Field>

{' '}

<Field name="confirm">
  <Description>
    Confirmation charges, for delivery signature, adult signature, etc.
  </Description>
</Field>

{' '}

<Field name="discount">
  <Description>A negative charge that is a discount for the user.</Description>
</Field>

{' '}

<Field name="fuel_charge">
  <Description>Fuel surcharges.</Description>
</Field>

{' '}

<Field name="additional_fees">
  <Description>
    Other fees that don't fit into the categories provided.
  </Description>
</Field>

{' '}

<Field name="tariff">
  <Description>Customs tariffs and levies.</Description>
</Field>

{' '}

<Field name="tax">
  <Description>
    Taxes relating to the shipment, like sales tax, VAT, etc.
  </Description>
</Field>

{' '}

<Field name="delivery">
  <Description>
    Fees associated with the delivery, appointment, COD, retry, etc.
  </Description>
</Field>

{' '}

<Field name="handling">
  <Description>
    For additionally handling, normally for non-machineable.
  </Description>
</Field>

{' '}

<Field name="special_goods">
  <Description>
    Fees for non-standard shipped goods, dangerous goods, fresh flowers, etc.
  </Description>
</Field>

{' '}

<Field name="pickup">
  <Description>For shipment pickup services fees.</Description>
</Field>

{' '}

<Field name="location_fee">
  <Description>
    Fees for rural areas, residential delivery, anything added as a result of
    the location.
  </Description>
</Field>

{' '}

<Field name="oversize">
  <Description>
    Fees related to the shape, weight, or size of the shipment.
  </Description>
</Field>

{' '}

<Field name="returns">
  <Description>Return fees stemming from a return label.</Description>
</Field>

{' '}

<Field name="notifications">
  <Description>
    Priority notifications from the carrier, opting into special notifications
    for services.
  </Description>
</Field>

</Reference>
